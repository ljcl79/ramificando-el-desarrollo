Dominando las Ramas en Git y GitHub

Aprende por qué y cómo usar ramas para un desarrollo más limpio, seguro y colaborativo.

¿Qué es una Rama?

Un Universo Paralelo

Imagina una rama como una copia de tu proyecto en un momento dado. En ella, puedes trabajar en nuevas funcionalidades, corregir errores o experimentar sin afectar la versión principal (main o master).

Trabajo Aislado y Seguro

El principal beneficio es el aislamiento. Si cometes un error grave en una rama, no pasa nada. La rama principal sigue intacta. Esto es crucial para el trabajo en equipo y para mantener la estabilidad del proyecto.

Organización Perfecta

Puedes crear ramas para cada tarea: una para "agregar-login", otra para "corregir-bug-header", etc. Esto organiza tu flujo de trabajo y deja un historial claro de cada cambio realizado en el proyecto.

Flujo de Trabajo Básico

1. Crear una Rama

Crea una nueva rama a partir de tu rama actual (generalmente `main`).

# Crea la rama y te mueve a ella
$ git switch -c nueva-funcionalidad

2. Trabajar y Confirmar

Realiza tus cambios: añade, modifica o elimina archivos y guarda tu progreso con commits.

# Haces tus cambios...
$ git add .
$ git commit -m "Agrega el formulario de login"

3. Subir a GitHub

Publica tu rama en el repositorio remoto para que otros puedan verla o para crear un Pull Request.

# Sube tu rama al origen remoto
$ git push origin nueva-funcionalidad

4. Fusionar (Merge)

Una vez terminada, fusionas tu rama con `main` para integrar tus cambios al proyecto principal.

# 1. Vuelve a la rama principal
$ git switch main
# 2. Fusiona los cambios
$ git merge nueva-funcionalidad

5. Eliminar la Rama

Después de fusionar, es una buena práctica eliminar la rama que ya no necesitas.

# Eliminar la rama local
$ git branch -d nueva-funcionalidad
# Eliminar la rama remota
$ git push origin --delete nueva-funcionalidad
